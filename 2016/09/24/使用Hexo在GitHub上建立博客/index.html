使用Hexo在GitHub上建立博客

发表于 09-24-2016   |   分类于 建立博客   |     |  

想写点文字，于是就开始搞博客。

去年我也是用的hexo，同时绑定了域名，然而发现自己很不经常写博客，所以一直就荒废了。前两天参加了团队所以就觉得要慢慢开始写东西了，于是又想起了博客。但是由于荒废的有点久，很多东西都忘记在哪里了，同时刚好NexT这个主题又进行了更新，所以我就打算从头开始搞起，也弥补一下我一年前搞完没写教程的遗憾。

网上有很多搭建博客的工具，但是找来找去还是觉得hexo比较方便，而且官方和民间都有很多主题可以用。创建博客不仅要有工具，你还要有你存储博客的地方，当然最好的是去买一个服务器用来存储，但是对于只写博客来说，专门去买一个服务器太浪费了。所以我用了GitHub本身就有的github.io来进行存储，它完全免费而且存储空间够用。
整体思路是这样的：

  1. 用hexo在本地创建库
  2. 将本地与Github关联起来
  3. 使用NexT主题来美化博客
  4. 开始写博客

下面是具体的内容

使用NexT美化博客

完成上述步骤后就已经完成了博客的建立，接下来我们来进行对博客的美化。
hexo官方提供了许多的主题可以使用，具体的你可以到这里进行挑选并应用到你的博客上面。

我使用的是由iissnan提供的NexT主题。该作者提供了一个专门的帮助网站来对博客进行美化，详情见NexT 使用文档，并不需要实现文档中所有的内容，挑你所好的就行，唯一要注意的是站点配置文件和主题配置文件不要搞错了。

开始写博客

使用hexo new <title>命令在_posts文件夹中创建新的文章，格式为markdown格式。有很多支持书写markdown文件的软件，我在用的是MWeb（Mac端）和TextNut（iPad端）软件。
同时你可以提前写好文章的模板，在博客库的scaffolds文件夹中创建一个新的markdown文件，这里我命名为blog.md，在里面写入你的模板内容就好了。要用这个模板创建文章只需使用hexo new blog <title>命令创建即可。
